# Phase 03: UI Components

**Goal:** Build every UI component following the HTML-First pipeline with plugin-assisted design. ONE component per session.

**Duration:** 10-20 sessions (shadcn handles foundation components)

---

## What's Different With Plugins
- **shadcn/ui provides foundation components** — Button, Input, Card, Badge, Avatar, Skeleton already installed from Phase 01
- **Frontend Design plugin** auto-activates — provides distinctive aesthetic direction
- **UI/UX Pro Max plugin** auto-activates — provides data-driven design recommendations
- **visual-diff-checker subagent** — validates HTML→React fidelity
- **code-reviewer subagent** — catches quality issues before commit

---

## The Pipeline (every component follows this)

### Step 1: Plugin-Assisted Design
CC detects a UI task → Frontend Design + UI/UX Pro Max auto-activate.
They provide aesthetic direction and specific recommendations.
CC grounds everything in PROJECT.md design tokens.

### Step 2: HTML Prototype
CC builds a standalone HTML/CSS file. Saved to: `design/prototypes/[component-name].html`

### Step 3: Developer Approval
You open the HTML file in a browser. Say "Approved" or give feedback.
CC does NOT proceed until approved.

### Step 4: React Conversion
CC converts to React/Next.js using shadcn/ui, Tailwind, Framer Motion.

### Step 5: Visual Diff
Invoke visual-diff-checker subagent. Compares at 375/768/1280.

### Step 6: Code Review
Invoke code-reviewer subagent. Checks quality, security, accessibility.

### Step 7: Commit
Only after both checks pass.

## Component Order

### Layout Components (sessions 1-4)
1. Header/Navbar (sticky, scroll-aware, mobile hamburger)
2. Footer (3-column grid, dark background)
3. Sidebar (if applicable)

### Feature Components (sessions 5-15+)
Build based on your PROJECT.md features. Examples:
5. ProductCard / ServiceCard
6. Grid / Listing layout
7. Filters / Search
8. Forms (using shadcn + react-hook-form + Zod)
9. [Continue based on your project]

### Page Compositions (last sessions)
Build pages LAST — they compose the components above.

## Session Template

```
Start of session:
  CC reads SESSION-STATE.md (or memory plugin injects context)

You say: "Build [ComponentName]"

CC:
  1. Frontend Design + UI/UX Pro Max auto-activate
  2. Reads PROJECT.md design system
  3. Builds HTML prototype → saves to design/prototypes/
  4. Shows you → waits for "Approved"
  5. Converts to React with shadcn/ui
  6. Runs visual-diff-checker subagent
  7. Runs code-reviewer subagent
  8. Commits
  9. Updates SESSION-STATE.md

You: /exit → fresh session → next component
```

---

## Rules for This Phase
- ONE component per session — no exceptions
- HTML prototype MUST be approved before React conversion
- Visual diff MUST pass before commit
- Code review MUST pass before commit
- Every component needs: loading skeleton, empty state, error state
- Use shadcn/ui components wherever possible
- Every image needs alt text
- Every icon button needs aria-label
- Animations need prefers-reduced-motion
- All colours from CSS variables — plugin suggestions must map to PROJECT.md tokens

## Definition of Done (per component)
- [ ] HTML prototype approved
- [ ] React component matches HTML at 375/768/1280
- [ ] shadcn/ui components used where appropriate
- [ ] Loading skeleton exported
- [ ] Empty and error states handled
- [ ] Accessibility checks pass
- [ ] Code review passed
- [ ] TypeScript clean
- [ ] Committed with descriptive message
